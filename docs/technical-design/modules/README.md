# Modules

## Purpose

This section groups Veil by implementation boundary. Every module page follows the same contract template so maintainers and contributors can read different modules without relearning the format.

## Read This Section If

- you are changing a specific module
- you need the module contract before reading code
- you want to understand cross-module dependencies

## Module Template

Each module page follows the same section order:

- Purpose
- Responsibility Boundary
- Out Of Scope
- Interface
- Data Flow
- State
- Errors
- Security Constraints
- Test Requirements
- Dependencies
