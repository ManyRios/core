# Glossary

## Purpose

This section defines the stable public terminology used across the Veil documentation set.

## Core Terms

| Term | Meaning |
|------|---------|
| gateway | the local OpenAI-compatible HTTP entry point used by Consumers |
| witness | a signed record used for usage evidence, export, and later reconciliation |
| discovery | the process of finding and selecting Relay nodes |
| accountless access | using Veil without binding every client directly to a dedicated upstream account flow |
| privacy-preserving routing | splitting visibility across roles without claiming perfect anonymity |
| capacity | usable execution supply exposed by Providers |
| quota | upstream account allowance or provider-side usage limit |
| settlement | turning request and witness evidence into billable or payable outcomes |
| build loop | the open-source contribution and review loop around Veil development |

## Writing Rule

- prefer the glossary term consistently across docs
- when a term is overloaded, define the local meaning explicitly
- avoid marketing synonyms that change the trust or product meaning
